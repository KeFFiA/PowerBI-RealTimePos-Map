# Aircraft Map Visual (Power BI custom visual)

Карта самолётов по широте/долготе на [Leaflet](https://leafletjs.com/): кастомные SVG-маркеры самолётов,
цвета по объектам, подписи, логотипы авиакомпаний, авто-зум, выделение областью (прямоугольник/полигон/лассо),
canvas-tooltips, локализация (en-US/ru-RU), landing page.

## Происхождение проекта

Исходный TypeScript-код был **утрачен** — остался только упакованный `AIXII_Aircraft.pbiviz`
(минифицированный webpack-бандл без source map). Проект восстановлен так:

- **Контракт и ассеты восстановлены 1:1** из бандла скриптом `tools/extract_from_pbiviz.py`:
  `capabilities.json`, `stringResources/**`, `style/visual.less`, `assets/icon.png`,
  `src/aircraftIcons.ts` (182 SVG-иконки самолётов).
- **Логика переписана с нуля** (`src/visual.ts`, `map.ts`, `markers.ts`, `selection.ts`,
  `dataModel.ts`, `settings.ts`) с тем же поведением; оригинальный бандл служил эталоном.

Файл `AIXII_Aircraft.pbiviz` оставлен в репозитории как референс — не удалять.

## Требования

- Node.js 18+ (рекомендуется 20+; часть dev-зависимостей предупреждает на Node 18, но сборка работает).
- Зависимости: `npm install`.

## Сборка

```bash
npm run package      # → dist/aircraftMapVisual....pbiviz
```

## Локальная отладка (`pbiviz start`)

```bash
npm start
```

⚠️ На Windows `pbiviz` создаёт dev-сертификат через **PowerShell 7 (`pwsh`)**. Если видите
`'pwsh' is not recognized` — установите PowerShell 7 и сгенерируйте сертификат один раз:

```bash
winget install --id Microsoft.PowerShell
npx pbiviz install-cert
```

Затем включите Developer Visual в Power BI Desktop/Service и перетащите визуал на холст.

## Поля данных (data roles)

| Роль | Назначение |
|---|---|
| ID / object name (`category`) | идентификатор объекта (обязательно) |
| Latitude / Longitude | координаты (обязательно) |
| Time / timestamp | время |
| Aircraft type field | код типа ВС (A320, B738, …) для выбора SVG |
| Label | подпись у маркера |
| Airline logo URL | URL логотипа авиакомпании |
| Tooltip | дополнительные поля для всплывающей подсказки |

Тестовый датасет: `sample-data/aircraft-sample.csv`.

## Настройки форматирования

- **Marker**: цвет по умолчанию, тип маркера (точка/самолёт), размер, подписи, логотип, размер логотипа.
- **Object markers**: индивидуальный цвет на каждый объект.
- **Map**: стиль подложки (Dark/Light/OSM/Color-Voyager), авто-зум.

## Соответствие оригиналу

Структура DOM и имена классов (`aircraft-map-root`, `aircraft-toolbar`, `aircraft-map-style-control`,
`aircraft-marker`, `aircraft-plane-shape`/`aircraft-shape-main`, …) и значения по умолчанию взяты
из оригинального бандла, поэтому восстановленный `style/visual.less` применяется без изменений:

- цвет маркера `#20d2c2`, маркер = «самолёт», размер `50`, подписи и логотип вкл., логотип `18`px;
- стиль карты по умолчанию `dark`, авто-зум вкл.;
- **тип ВС по умолчанию `B738`** (когда поле типа и пресет не заданы) — поэтому самолёты видны
  даже без поля «Aircraft type»;
- иконки тулбара, трансформация масштабирования самолёта (×1.46) и подсветка выбора (`#22d3ee`) — как в оригинале.

## Известные ограничения / открытые вопросы

- **ru-RU не переведён** — в оригинальном бандле русская локализация была идентична английской.
  Восстановлено 1:1; перевод строк — отдельная задача (`stringResources/ru-RU/resources.resjson`).
- **Цвет**: по умолчанию `#20d2c2` для всех, индивидуальный цвет на объект — через «Object markers»
  (как в оригинале: `objectMarker.fill` переопределяет дефолт).
- **Алиасы типов ВС**: прямой поиск по коду (A320, B738, …) и 182 SVG восстановлены; таблица
  алиасов (например `A320NEO`→`A20N`) из оригинала пока не извлечена — при необходимости добавим.
- **Геометрия лассо/полигона/прямоугольника** считается в гео-координатах (как в оригинале).
